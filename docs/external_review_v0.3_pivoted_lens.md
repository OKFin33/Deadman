# Deadman 评测补正 v0.3 — 陪伴搭子/不吐不快/轻度 if 线讨论总领

> 评测日期：2026-06-03  
> 文档性质：**对 v0.2A 和 v0.2B 的反向补正**——v0.2A/2B 默认 Deadman 是 "RPG 系统"，但产品方所确认的实际总领是 **"陪伴搭子 + 看剧过程中不吐不快的承接 + 轻度 if 线讨论"**。v0.2A 中"展示 consequence 长文 / 暴露 canon_anchor 对照 / 暴露 verdict.label 立场徽章"等 3 项推荐与该总领相反，本文件作正式作废 + 修正。  
> 适用范围：**v0.3 是当前最新版**，覆盖 v0.2A/2B 中与总领冲突的部分。v0.2A/2B 关于代码-产品差距测试方法、catalog 单卡问题、分享按钮黑洞、继续看不自动播放等仍然成立的发现继续生效。

---

## 0. TL;DR

**v0.2A 把 Deadman 当成"严肃 AI 判定系统"，推荐了"展示后端深度"方向。这是错方向。**

Deadman 的产品总领是**"轻度陪伴 + 吐槽承接 + if 线讨论"**——一个让你看剧时偶尔想说一句的搭子，不是一个对你的选择下结论的评委。

**修正后核心结论**：

| 维度 | v0.2A 的判断 | v0.3 修正 |
|---|---|---|
| 代码-产品差距 | 13% payload 暴露 = 巨大浪费 | 13% 暴露**有大半是对的**——后端深度服务于 quality/governance，前端轻度服务于陪伴。不是 gap，是分层 |
| 结果文案太短 | 应该展开 consequence 长文 | **保持单行**，但文风从"verdict"改"搭子 echo" |
| canon_anchor 对照 | 应该暴露做"原剧 vs 你" | **不暴露**——会让搭子变教科书 |
| verdict.label 立场 | 应该做立场徽章 | **不做**——会让搭子变裁判 |
| 真正的产品 gap | 评判式偏深 | **陪伴姿态偏 reactive、缺主动温度** |

## 1. 总领重述

产品方确认的 Deadman 总领（粘贴自对话）：

> "Deadman 在此前转向中，强调陪伴、搭子、情绪价值，是为看剧过程中，偶尔可能出现的观众对情节不吐不快的承接，是对吐槽、if 线的轻度讨论。"

拆解：
- **陪伴** > 系统：搭子在场是核心，不是 AI 系统在评判
- **情绪价值** > 决策价值：用户来这里是要被接住情绪，不是被打分
- **不吐不快的承接** > 决策邀请：marker 命中不是"请你做决定"，是"如果你想说我接着"
- **轻度** > 深度：保持薄、保持快、保持可路过
- **吐槽 + if 线讨论**：两种动作，吐槽更高频更轻、if 线讨论稍重一点

**实质上，Deadman 在产品类型上更接近"弹幕替代品 + 搭子 chat 气泡"，不是"互动剧引擎"**。

## 2. v0.2A 的反向作废清单

以下 3 项推荐**正式作废**，理由：与新总领的"轻度"原则反向。

### ❌ 作废 1：把 `verdict.consequence.text` 长文展示出来
- v0.2A 理由：让 AI 看起来在认真工作
- 总领冲突：**长文＝评委腔**。consequence 那种"'要是我来...'站得住，但要按眼前尺度执行..."的句子，在搭子产品里是把朋友变知乎答主
- 正确做法：单行响应保留，但文风改"搭子 echo"（见下文 Gap 2）

### ❌ 作废 2：暴露 `canon_anchor.original_plot_note` 做"原剧 vs 你"对照
- v0.2A 理由：天然的差异化展现 frame
- 总领冲突：**对照＝教科书**。把"原剧把兔子处理成家人信任修复的一步"摆出来，搭子就变成了"剧情研究员"，破坏陪伴姿态
- 正确做法：canon_anchor 留在 payload 里作为**判定质量保险**（防止搭子说错话），但**不暴露给 UI**。这反而是"代码做了产品不暴露"的正确分层

### ❌ 作废 3：暴露 `verdict.label` + `stance` 做立场徽章
- v0.2A 理由：让评委看到 AI 在下结论
- 总领冲突：**立场徽章＝裁判**。"稳，但别摊太大 · 搭子支持"这种 chip 是审判而不是陪伴
- 正确做法：stance/label 留 payload 内部驱动文案生成（决定搭子说什么），但**不作为可视元素**

**结论**：v0.2A 的 8 项推荐里前 3 项都因为同一原因作废——**把产品 UI 当成了暴露后端能力的橱窗**，但 Deadman 不是橱窗类产品。

| 结果"我接住你咯 + 单行" | ⚠️ 单行太短不像 AI 在判 | ✅ **改判**——单行是对的，但文风要从"verdict"改"echo" |
| "继续看 ▶" 不自动播 | ⚠️ 半承诺 | ⚠️ **保留**——和总领高度冲突（陪伴产品不该卡 flow） |

### 仪器 2：Payload 可见度地图（在新总领下重读）

新总领下，"暴露率低"不再普遍是问题，要按字段分类：

**应该保留隐藏（隐藏是对的）**：
- `judgment.consequence.text` —— 太重
- `judgment.canon_anchor.*` —— 教科书化
- `judgment.scores` (6 维) —— 评委化
- `judgment.judgment_basis.*` —— 内部质量保证
- `judgment.engine.mode` —— 内部
- `judgment.media.status` —— 内部

**应该用，但用法和 v0.2A 不同**：
- `judgment.verdict.label` / `stance` —— ❌ 不做徽章。✅ **作为搭子文案库的索引**：stance=support → "啊就是""我懂你"；stance=oppose → "等等先别"；stance=neutral → "你说，我想想"。**驱动文案，不显化为 chip**
- `judgment.verdict.summary` —— ✅ 保留单行展示，但文案池要从"verdict"改"echo"
- `session_memory.last_choice_summary` —— ✅ **真亮点**。新总领下不是"被严肃记住"，是"搭子还记得"。下次同剧切回来时搭子说"上次你说……"，是陪伴感的最重要落点
- `companion.utterance` —— ⚠️ **现在永远为空字符串**。这个字段就是给"搭子主动说一句"用的，前端没接（见 Gap 1）
- `companion.should_interrupt` —— ⚠️ 现在通过 pause 体现，新总领下应该更显化：搭子打断 vs 搭子飘过，对应不同的姿态

**应该新做（产品交互层）**：
- "再吐一句"入口（payload 已支持多次 judgment）
- "路过"姿态（用户不动作时搭子自己收手 + 留一句）
- 吐槽 vs if 线分层入口

### 仪器 3：Emotional arc（在新总领下重画）

v0.2A 期望的曲线（评委视角）：
```
缓爬升 → 选择 → 高峰（深度反馈）→ 平
```

新总领下应该是的曲线（搭子视角）：
```
微起波（marker 出现）→ 路过或承接 → 小回响（搭子 echo）→ 回到剧
（如果想再吐一次：小波澜重来）
```

差异：
- **峰值高度**：v0.2A 期待高峰；新总领期待微波 + 多次重现
- **频次**：v0.2A 单次 loop；新总领多次轻量重入
- **退出**：v0.2A 期待"结果让你 satisfied"；新总领期待"被接住后顺滑回剧"

### 仪器 4：盲读（新总领下重读）

同样 5 张截图：

- 截图 1（catalog）：新总领下仍弱——文案不像"陪你看剧"，像"产品介绍页"
- 截图 2（player 首屏 "!"）：搭子在角落举着 "!" 不动——新总领下应该是搭子自己嘀咕一句、然后等你
- 截图 3（选项面板）：3 个口语化选项 + 自由输入——在 if 线讨论场景下是**对的**；但门槛对纯吐槽冲动来说**太高**
- 截图 4（结果面板）：搭子举 "懂你❤" + 单行——**新总领下方向对**，但文案 "这步可以，爽点不是把后面全改了" 仍偏 verdict 调
- 截图 5（episode dropdown）：5 集横陈——OK，但每集 hook 文案仍冷静

**盲读结论修正**：v0.2A 说"产品故事是断的"——在 RPG 系统视角下成立；在搭子视角下，**故事不是断的，是搭子不够 alive**。

### 仪器 5：Elevator pitch（新总领下重写）

v0.2A 评委复述（旧）：
> "看短剧的时候有个橘色搭子陪你，关键剧情点会问你'要是我来你会怎么做'，三个选项 + 自由输入，AI 给你一句回应。挺可爱的。"

新总领下**应该让评委复述**：
> "看剧时你偶尔想吐一句，旁边有个搭子真的接住你。你说啥它都顺着说，但你要认真讨论它也能陪你聊几句。"

**Gap**：现状离这个目标 60-70% 距离。要把"接住情绪"这件事做得**比当前更主动、更轻、更可重复**。

## 4. 新总领下的 5 个真问题

替换 v0.2A 第 6 节的 6 个"代码-产品差距"问题——这些是 v0.3 视角下的真 gap：

### Gap 1：搭子是 reactive 不是 alive
- 现象：搭子只在用户主动点击或 marker 命中时换状态。1:06 的视频里搭子是冻结的角色海报
- 证据：`companion.utterance` 字段在所有 payload 里都是空字符串——后端预留了，前端没接
- 影响：陪伴感停在"在场"，没有跨到"在身边"
- 修法：在视频播到某些段落时由后端推 utterance（"这家一年没吃过肉啊..." / "这眼神看着难受"），前端用气泡显示 2-3 秒自动消失

### Gap 2：结果回弹太"verdict"调
- 现象："我接住你咯 + 这步可以，爽点不是把后面全改了..." 是评委腔
- 影响：朋友突然变成知乎答主
- 修法：**保留单行体量**（不展开 consequence），但**重写 result_surface 文案池**：
  - 不要："这步可以"/"稳，但别摊太大"
  - 要："啊就是" / "看到那个眼神我也忍不了" / "这家人确实需要这一下"
- 实现：用 `verdict.stance` 当索引，stance=support 走"附和式"，stance=oppose 走"打断式"，stance=neutral 走"陪你想式"

### Gap 3：缺"路过"通道
- 现象：marker 命中 → 搭子举着 "!" 不动 → 用户不点就一直举着
- 影响：把"如果你想说"变成了"你必须说"
- 修法：搭子 notice 状态进入 N 秒（5-8 秒）后如果用户没动作，搭子自己收手 + 留一句飘过气泡（"算了等下还会再有"），回到 idle。**这给搭子一个"主动放手"的姿态**，比死等温柔

### Gap 4：吐槽天然重复，没有"再吐一句"入口
- 现象：选完一个选项 → 看结果 → 关闭 → 回剧。一次性
- 影响：第二、第三句吐槽冲动没有承接口
- 修法：结果面板上加"还想说"按钮，回到自由输入。或者搭子在结果面板说完 echo 之后追加"还有要说的吗？"

### Gap 5：吐槽 vs if 线讨论没分层
- 现象：所有 marker 都通向同一个面板（hook + 3 选项 + 自由输入）
- 影响：想吐一句"😅 离谱"的人觉得门槛高；想认真讨论的人会用，但前者流失
- 修法：marker 命中默认弹**轻量气泡**（3 个 1-tap 表情/短词反应，比如"😅 离谱""💔 心疼""🔥 就是啊"），点完直接回剧；用户想深入再点"展开 if 线..."按钮，才走到现在的选项面板。**吐槽路径变薄，if 线路径留厚**

## 5. 新总领下的 8 天补救清单

按"对陪伴感/吐槽流畅度的边际效用"排序：

### 优先级 1：接 `companion.utterance` + 后端推 2-3 句段落话（< 4h）
- 前端：companion 加 utterance 气泡渲染 + 3 秒淡出
- 后端：在 viewer_session/event 响应里偶尔携带 utterance（不需要每次）
- **这一项是 Gap 1 的解，也是新总领下最值得做的一件事**

### 优先级 2：result_surface 文案池改"搭子 echo"风（< 3h）
- 重写当前的 verdict 文案模板
- 接入 `verdict.stance` 做文案分支
- 保持单行体量

### 优先级 3：吐槽 vs if 线分层入口（< 8h）
- marker 命中默认弹 1-tap 气泡（3 个轻反应）
- 加"展开 if 线..."按钮通向现在的完整选项面板
- 这一项虽然贵，但是 Gap 5 的解，直接拉低吐槽门槛

### 优先级 4：修分享按钮，做"搭子聊了一句"卡片（< 4h）
- 分享对吐槽产品是核心出口
- 卡片内容：搭子 + 用户那句 + 一句 echo + 二维码占位
- 截图友好

### 优先级 5：catalog 文案 + hero 软化（< 6h）
- "5 个已发布介入点" → "今天 5 个想吐的瞬间" 或类似
- 主标题加 tagline：例如"看剧也想说一句？搭子在听"
- Hero 占位换成剧情触发瞬间的高对比静帧（不必动效，但要"有事发生"感）

### 优先级 6：搭子"路过"姿态（< 3h）
- notice 状态 5-8 秒无动作 → 自己收手 + 飘一句话
- 实现简单：companion state machine 加 idle-from-notice 转移

### 优先级 7：last_choice_summary 用作"搭子还记得"（< 3h）
- 同剧切回时搭子说"上次你说……"
- 注意文风必须是"还记得"不是"已记录"

### 优先级 8：继续看自动恢复播放 + Branch 3 命名清理（< 1h 合计）
- 小修

**全部 < 32 小时，8 天内 comfortable。**

**关键变化对比 v0.2A**：
- 删除：consequence 展开 / canon_anchor 对照 / verdict.label 徽章
- 新增：搭子 utterance / 路过姿态 / 吐槽 if 线分层 / 搭子 echo 文风
- 升级优先级：分享按钮（从 P6 到 P4）
- 保留：catalog / 自动播放 / Branch 3 命名

## 6. v0.2B 部分调整

v0.2B 的 C 端视角判断需要**部分降调**：

**仍然成立**：
- 短剧用户爽点节奏（30s/次）和 Deadman 互动节奏（1-2 分钟/次）的错位——v0.3 的"吐槽 vs if 线分层"是这个错位的直接回应：吐槽路径可以做到 5-10 秒/次，接近短剧节奏
- 题材选择（荒年苦难）和 C 端短剧主流题材不匹配——不变
- 自由输入打字成本高——v0.3 的轻量气泡入口正是对这一点的修正

**需要降调**：
- v0.2B 判定"产品定位错位需要换 target 用户"——**过度了**。新总领下 Deadman 是 broadcast 式 + 个人化的弹幕替代品，C 端是可能的目标，**关键是搭子姿态足够轻**
- v0.2B 推荐"路线 A vs 路线 B"二选一——**也过度了**。新总领下 Deadman 已经在走"轻度陪伴 + 吐槽承接"路线，不需要二选一

**v0.2B 的"3.5/10 上线适配度"打分要重打**：在新总领下，如果 v0.3 优先级 1-3 落地，**预期能到 5-6/10**，进入"可上线试验"区间，不再是"结构性错位"区间。

## 7. 给答辩的反向利用（修正版）

v0.2A 建议的答辩演示脚本（"展示 governed runtime / canon-safe / 原剧对照"）**作废**——这是评委向的橱窗，不是产品向的展示。

新总领下的答辩演示脚本应该是：

1. catalog → 看到"看剧也想说一句？搭子在听" → 评委 get 到定位
2. 进 EP12 → 搭子在角落，过 5 秒搭子自言一句"这家一年没吃过肉啊..." → 评委看到陪伴
3. marker 命中 → 弹 1-tap 气泡"😅 离谱""💔 心疼""🔥 就是啊" → 评委看到轻吐槽路径
4. 演示者点"展开 if 线..." → 走完整选项 → 评委看到深路径
5. 选完 → 搭子 echo "啊就是，这家确实需要这一下" → 评委看到陪伴回应（不是评委判定）
6. 演示者点"还想说" → 自由输入 → 搭子又 echo 一次 → 评委看到吐槽是可重复的
7. 切回 EP12 → 搭子说"上次你说……" → 评委看到陪伴是有记忆的
8. （可选）演示者切到 EP03 → 看到 "?" 憋住点 → 评委看到两种情绪模式

**答辩主叙事**：
> "我们不做评委式 AI——我们做一个看剧时偶尔接住你一句话的搭子。轻、可路过、可重复、还记得你。后端的 governed runtime 不是用来炫的，是用来保证搭子说话不会脱离剧情边界。"

这段把"代码深度"翻成"陪伴保障"——既是真话，也匹配总领。

# Branch 3 Demo Episode Pack Contract v0.1

This is the minimum content contract for the lightweight Branch 3 demo player.
The frontend should be able to assemble an episode surface from imported pack
data without hardcoding which four or five episodes are used.

## Source Fields

Each selectable episode is derived from `DeadmanMomentSummary` entries returned
by `/api/deadman/dramas/{dramaId}/moments`.

Minimum fields per moment:

- `source_drama.episode_id`: groups moments into one selectable episode.
- `source_drama.runtime_video_url`: video file URL for that episode.
- `source_drama.title`: displayed drama title when present.
- `interaction_window.notice_at_seconds`: when the companion first notices.
- `interaction_window.start_seconds`: start of the tappable window.
- `interaction_window.end_seconds`: end of the tappable window.
- `notice_marker`: `!` or `?` companion emphasis.
- `hook`: emotional cue / companion prompt seed.
- `action_type`: maps to the voice question style.
- `default_options`: three quick replies.
- `result_media`: optional reserved result asset slots.

## Runtime Behavior

- The episode picker is generated by grouping imported moments by
  `source_drama.episode_id`.
- Switching episodes resets playback time, completed markers, notified markers,
  custom input, result state, and companion state.
- The current episode only renders markers from its own `episode_id`.
- Notice state does not pause playback; opening choices pauses playback.
- Cancelling a bubble can reopen while still inside the active window.
- Submitting a choice marks that moment completed, so it cannot reopen from idle
  in the same watch pass.

## Demo Authoring Rule

To add or swap a demo episode, import a video and one or more moment entries
with the fields above. The UI should not need code changes unless the data
schema changes.
